## 🎯 Our Standards

We are committed to maintaining a **welcoming, inclusive, and professional** community. All contributors, maintainers, and users are expected to:

### ✅ Expected Behaviors

| Behavior | Description |
|----------|-------------|
| **Professional Communication** | Engage respectfully and constructively in all interactions |
| **Evidence-Based Review** | Review technical claims against facts, documentation, and tests |
| **Credential Protection** | Never expose, request, or misuse API keys or credentials |
| **Personal Media Protection** | Respect privacy; never share others' personal media without consent |
| **Lawful Conduct** | Respect all applicable laws and regulations |
| **Consent Requirements** | Honor consent requirements, especially for face/media processing |

### ❌ Unacceptable Behaviors

| Behavior | Status |
|----------|--------|
| Harassment of any kind | ❌ Not Accepted |
| Threats or intimidation | ❌ Not Accepted |
| Doxxing (sharing private information) | ❌ Not Accepted |
| Discriminatory conduct | ❌ Not Accepted |
| Malicious credential collection | ❌ Not Accepted |
| Non-consensual likeness workflows | ❌ Not Accepted |
| Deliberate concealment of security defects | ❌ Not Accepted |
| Hate speech or offensive language | ❌ Not Accepted |

> These standards apply to all project spaces, including GitHub issues, pull requests, discussions, and any other communication channels.

---

## 🛡️ Our Responsibilities

Project maintainers are responsible for clarifying and enforcing our standards of acceptable behavior. We have the right and responsibility to:

- Remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that do not align with this Code of Conduct
- Temporarily or permanently ban any contributor for other behaviors that we deem inappropriate, threatening, offensive, or harmful
- Interpret these guidelines as we see fit, including the decision to make exceptions

---

## 📋 Enforcement

### Reporting Violations

If you experience or witness unacceptable behavior, or have any other concerns, please report it by:

1. **Contacting maintainers privately** through the security contact configured for the repository
2. **Using GitHub's reporting tools** for public content violations
3. **Including as much information as possible:**
   - Who was involved
   - What happened
   - When it happened
   - Where it happened (GitHub issue, PR, discussion, etc.)
   - Screenshots or links to the content (if applicable)

All reports will be reviewed and investigated promptly and fairly. Confidentiality will be maintained to the fullest extent possible.

### Enforcement Process

1. **Report Received** - Maintainers acknowledge receipt within 24 hours
2. **Investigation** - Fair and thorough review of the report
3. **Response** - Maintainers determine appropriate action
4. **Resolution** - Action is taken and communicated to relevant parties

Maintainers who do not follow or enforce the Code of Conduct in good faith may face temporary or permanent repercussions as determined by other members of the project's leadership.

## 🌍 Scope

This Code of Conduct applies within all project spaces, including but not limited to:

- GitHub repository (code, issues, pull requests, discussions)
- Project documentation
- Community forums and chat rooms
- Social media under project accounts
- Any other venues used for project business
